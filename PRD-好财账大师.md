## 1. 概述

### 1.1 产品背景

中小微企业及代理记账机构在做账报税环节普遍面临：票据分散、手工凭证易错、发票/银行/工资数据相互割裂、三大税种测算依赖会计经验、申报表填报繁琐。好财账大师以 **"AI 三步做账报税"** 为核心方法，将报税流程标准化为「员工工资 → 发票采集 → 银行流水 → 数据核对 → 生成申报」五步，通过 AI 辅助完成税额测算、异常定位与报税总结，降低专业门槛与人工成本。

### 1.2 产品目标

| 维度 | 目标 |
| --- | --- |
| 用户价值 | 让非专业财务也能独立完成月度/季度报税全流程，专业会计效率提升 |
| 业务目标 | 验证"三步报税"主流程跑通，完成需求确认与流程定稿 |
| 一期边界 | 流程展示与需求确认为主，不接入真实税务局/银行接口 |

### 1.3 名词与角色

| 名词 | 说明 |
| --- | --- |
| 报税工作台 | 登录后首页，集中展示进度、指标、税额与待办 |
| 三步报税 | 工资 → 发票 → 银行流水 三步基础数据采集 |
| 销项/进项发票 | 销售开出的发票 / 采购取得的发票 |
| 费用小票 | 无增值税票样的消费凭证（餐饮、停车、快递等） |
| 区块链发票 | 深圳区块链、浙江通用电子发票等正规电子发票 |
| 无票支出 | 未取得发票但实际发生的支出，可入内部账 |
| 零申报 | 无发票、无银行流水、有员工信息且工资为 0 时的简化申报模式 |
| 三大税种 | 增值税、企业所得税、个人所得税（含分红个税） |
| 申报期 | 月报企业按月，季报企业按季度 |

**典型角色**

| 角色 | 使用终端 | 主要职责 |
| --- | --- | --- |
| 企业主 / 经营者 | 移动端为主 | 掌握报税进度、查看税额、催办待办 |
| 企业财务 / 出纳 | PC 为主、移动端辅助 | 录入工资、发票、流水，核对并生成申报 |
| 代理记账会计 | PC 后台 | 管理多套企业账套，批量处理 |
| 被授权人 | 移动端 | 接受公司授权，操作指定公司账套 |

## 3. 详细功能需求

### 3.1 账号与企业

#### 3.1.1 注册 / 登录（双端）

- **入口**：
  - 移动端：首次打开 App 进入登录页，支持"登录 / 注册"切换。
  - PC 后台：访问 `/sign-in` 进入登录页，同样支持注册。
- **登录方式**：
  - 手机号 + 6 位短信验证码。
  - 账号密码登录（PC 与移动端均支持）。
- **校验规则**：
  - 手机号：`/^1\d{10}$/`（11 位）。
  - 验证码：6 位纯数字。
  - 密码：6–32 位字符。
  - 验证码按钮：60s 倒计时，倒计时期间禁用，文案"X 秒后重发"。
- **协议确认**：登录/注册即视为同意《用户协议》《隐私政策》（底部固定提示）。
- **密码管理**：已登录用户可在设置中修改/设置登录密码。
- **原型说明**：验证码仅为模拟，不发送真实短信。一期接入真实短信网关为待定项。

#### 3.1.2 企业上下文

- 顶栏 / 侧栏固定展示**当前服务企业**（如"北京好财科技有限公司"）+ 纳税人类型 + 当前申报期（如"一般纳税人 · 2026 年 6 月申报期"）。
- PC 顶栏提供"切换企业""AI 财税助手""进入申报"快捷操作。
- 移动端顶栏待办图标展示待处理事项数量（如 3 条）。

#### 3.1.3 多企业管理与授权

| 功能 | 说明 |
| --- | --- |
| 切换公司 | 弹层列表，支持搜索、选择当前企业，可新增企业 |
| 新增企业（创建公司） | 填写：税号（统一社会信用代码）、企业名称、地区（省级行政区下拉）、纳税人类型、申报周期、经办人手机号、密码；校验完整后创建 |
| 编辑企业 | 修改企业名称、税号、地区、纳税人类型、申报周期、电子税务局账号/密码等 |
| 公司授权（我的 → 公司授权） | Tab：**已授权**（我授权给他人） / **被授权**（他人授权给我）；公司列表内可直接添加授权手机号，授权他人操作该企业 |
| 添加授权 / 撤销授权 | 选择被授权公司 + 填写已注册手机号，确认后生成授权记录；授权人可随时撤销授权 |
| 授权记录字段 | 被授权手机号 / 被授权人姓名 / 授权公司 / 授权时间 / 状态 |
| 企业付费状态 | 创建/切换企业时提示是否已付费；未付费企业限制使用核心报税功能 |
| 公司列表筛选 | 支持按企业名称、纳税人类型、地区、付费状态、有效期等条件筛选与排序 |

> 验收要点：授权对象必须为已注册手机号；授权后被授权人可在其账号下操作对应公司账套。

#### 3.1.4 付费套餐、专属客服与公司列表筛选

- **付费套餐**：
  - 根据企业类型/规模推荐套餐：普通版（月付）、一年版、终身版。
  - 套餐展示价格、有效期、包含功能、最大账套数等。
  - 创建企业后提醒付费；未付费企业仅可查看基础信息，不可生成申报/提交。
  - 支持模拟支付（一期演示），支付成功后更新有效期与付费状态。
- **发票申请**：
  - 购买套餐后可申请开具发票，填写发票抬头、税号、邮箱。
  - 后台记录开票状态，支持查看开票历史。
- **专属客服**：
  - PC 后台设置/帮助入口提供“专属客服”按钮。
  - 点击后弹出企业微信二维码，用户扫码添加客服咨询。
- **公司列表筛选**：
  - 当企业数量较多（如 100 家）时，支持按名称关键字、纳税人类型、地区、付费状态、有效期等条件筛选与排序。
  - 列表展示付费状态标签（已付费 / 未付费 / 已过期）。

### 3.2 报税工作台（首页）

**目标**：一屏掌握本期报税进度、数据完整性、税额与待办。

#### 3.2.1 PC 后台布局（自上而下）

1. **报税进度卡**：五步流程节点，状态色区分（done / current / warn / 待办）。
   - Step1 员工工资 / Step2 发票采集 / Step3 银行流水 / Step4 数据核对 / Step5 生成申报。
   - 顶部展示整体进度百分比（如 72%）。
2. **核心指标卡（5 列）**：员工工资（人数）、发票采集（张数/异常）、银行流水（条数/待匹配）、数据核对（通过/总数）、预计应缴税额（环比）。
3. **三大税种测算卡（3 列）**：增值税 / 企业所得税 / 个人所得税，展示金额 + 关键明细 + 跳转入口。
4. **待处理事项 + AI 核对建议**（双列）：红/橙/绿点分级提醒与 AI 建议。

#### 3.2.2 移动端布局

- 页面标题区：报税工作台 + 副标题（"本期申报数据已准备 72%，还有 3 个待办事项"）+ 刷新。
- 进度卡（含进度条 + 五点步骤条）。
- 指标网格（2 列，含 wide 卡显示预计应缴税额）。
- 快捷处理（4 宫格：工资 / 发票 / 流水 / 核对）。
- 税额测算（横向滚动 mini 卡）。
- 待处理事项列表。

#### 3.2.3 字段与规则

| 数据项 | 取数口径 |
| --- | --- |
| 整体进度 | 已完成步骤 / 5（或按数据完整度加权，一期可固定展示） |
| 员工工资 | 当期已确认员工人数 + 工资总额 |
| 发票采集 | 销项张数 / 进项张数 / 异常张数 |
| 银行流水 | 本期流水条数 / 待匹配条数 |
| 数据核对 | 已通过项 / 总核对项（如 8/10） |
| 预计应缴税额 | 三大税种合计（一期 42,610），含环比 |
| 待办 | 未认证进项、未匹配流水、待确认无票支出等，按"影响申报"优先级排序 |

### 3.3 三步报税

#### 3.3.1 流程总览与零申报

- **入口**：导航"三步报税"，PC 为三步卡片 + 后置流程；移动端为模块列表。
- **零申报开关（移动端）**：
  - 开启条件（全部满足）：无发票往来、无银行流水、有员工信息、员工工资为 0。
  - 开启后系统提示风险："若产生开票或银行付款业务，可能导致账务税务问题，平台不承担相关责任"。
  - 二次确认弹窗："知悉并开启" / "暂不开启"。
- **下一步建议**：先处理异常发票与待匹配流水，再进入数据核对。

#### 3.3.2 Step 1：员工工资

**列表/表格**

- 支持按月份切换（2026-05 / 06 / 07）。
- 指标：在职员工数、工资总额、社保公积金合计（个人+公司）、预计个税。
- 列：员工姓名、身份证后四位、手机号、应发工资、个人社保、个人公积金、专项扣除、应扣个税、公司社保成本、公司公积金成本、企业用工总成本、状态（在职/离职 · 已确认/待确认）、操作（修改/删除/查看）。
- 操作：新增员工、导入工资表（一期为演示入口，不读取文件）、确认本月工资、删除（二次确认）。
- **工资历史**：支持按员工/期间查询历史工资记录，展示应发、社保公积金（个人+公司）、个税、实发工资、企业成本。

**员工详情/表单（字段）**

| 分区 | 字段 |
| --- | --- |
| 基础信息 | 姓名、证件类型（身份证/港澳居住证/护照）、证件号码、国籍、性别、手机号、入职时间、任职部门（下拉选择）、岗位、员工状态（在职/离职） |
| 工资与个税 | 应发工资、专项附加扣除、计税月份 |
| 五险一金 | 各险种缴纳开关、缴纳模式（按基数 / 按金额）、缴纳基数/金额、个人比例、公司比例 |

**社保公积金配置规则**

- 社保、公积金比例不固定，新增/编辑员工时可按实际参保方案逐项调整。
- 支持险种：养老保险、医疗保险、失业保险、工伤保险、生育保险、大病医疗、住房公积金。
- 每险种可独立设置：是否缴纳、缴纳基数/金额、个人比例、公司比例、启用状态。
- 缴纳模式支持“按基数 × 比例”或“直接填写金额”两种方式切换。
- 公积金比例可选 5% / 7% / 8% / 10% / 12% 或自定义。
- 支持批量设置基数、清空、全选等快捷操作（PC）。

**个税测算（实时刷新）**

- 应税收入 = max(0, 应发工资 − 个人社保 − 个人公积金 − 专项扣除 − 起征点 5000)。
- 个税按 7 级超额累进（3%–45%，速算扣除数）。
- 同时输出个人社保、个人公积金、应扣个税、实发工资、公司社保成本、公司公积金成本、企业用工总成本。

> 移动端为轻量字段（姓名 / 身份证 / 手机号 / 部门 / 应发 / 专项扣除 + 社保公积金配置 + 测算结果 + 在职/离职状态），PC 为完整字段。

#### 3.3.3 Step 2：发票采集与管理

**PC 后台**

- 指标卡：销项金额 / 进项金额 / 费用票金额 / 无票支出 / 异常发票。
- Tab 筛选：全部 / 销项 / 进项 / 费用 / 异常 / 无票支出；月份筛选；入账方式筛选。
- 表格列：发票号码、开票日期、往来方、类型、金额（不含税）、税额、税率、价税合计、费用归类、认证状态、入账状态、风险状态、操作。
- 操作：
  - **自动采集**：点击后返回采集结果提示，包含采集发票张数、销项金额/张数、进项金额/张数、异常张数等汇总数据。
  - **上传发票**：支持上传 PDF/图片发票文件（一期可先保存文件 + 人工补录，OCR 识别为后续项）。
  - **无票支出**：按钮名称为“无票支出”，点击录入无票支出（字段：员工、金额、日期、费用归类、说明；不含税率，不关联银行流水）。
  - 发票处理 / 红冲。

**移动端（按季度/月份采集）**

- 年度发票统计 + 季度 Tab（一/二/三/四季度）。
- 季度汇总：销项金额、进项金额、费用小票、无票支出、异常发票。
- **每月采集卡**：
  - "立即采集 / 重新采集"按钮 + 采集状态（未采集 / 已采集）。
  - 采集完成后提示本月采集结果（销项/进项张数与金额）。
  - 销项发票 / 进项发票 卡片（金额 + 张数，点击查看明细）。
  - **温馨提醒**："系统不会自动采集当月发票数据，需自行发起采集；数据截至最近一次采集时间。"
- **手动上传区**：
  - 费用小票：餐饮住宿等，支持混贴；展示金额/张数。
  - 区块链发票：深圳/浙江电子发票，单张拍照，不支持混贴；展示销项/进项金额与张数。
  - 无票支出：按员工记录，含员工姓名、金额、日期、费用归类（交通/餐饮/住宿/办公/差旅/其他）、说明；展示已记录列表（按员工分组汇总）。

**拍照识别上传（移动端专属）**

- 票据类型二选一：**费用小票**（混贴可多张）/ **区块链发票**（单张，需票面清晰）。
- 拍照 / 相册上传两个入口。
- 统计区：上传份数 + 合计金额（小票）；销项/进项份数与金额（区块链）。
- 一期为模拟识别，真实 OCR 为后续接入项。

**发票状态枚举**

| 维度 | 取值 |
| --- | --- |
| 认证状态 | 已认证 / 待认证 / 无需认证 |
| 入账状态 | 已入账 / 待确认 / 待匹配 |
| 风险状态 | 正常 / 异常 / 已红冲 |
| 类型 | 销项 / 进项 / 费用 / 无票支出 |

#### 3.3.4 Step 3：银行流水

- 支持多账户（如基本户 招行 8898、一般户 工行 2391），每账户展示尾号、本期流水条数、待匹配条数。
- **添加银行账户**：
  - 账户类型（基本户 / 一般户）放在表单顶部。
  - 开户银行为下拉选择（支持常见银行，亦可手动输入）。
  - 字段：账户类型、开户银行、账号、账户名称。
- **流水明细**：
  - 默认按当期申报期查询。
  - 列：交易日期、银行账户、对方户名、收入/支出金额、摘要、匹配发票、入账方式（按账期入账 / 全部入本账期）、状态（已匹配/待确认/待匹配）、操作（匹配）。
  - 筛选条件：月份、入账方式、银行账户（增加“全部”选项）。
- 操作：添加银行账户、上传流水（一期演示/占位）、银行直连（一期占位）、单笔匹配、批量匹配。

> **银行直连**：原型明确标注"暂未接入银行接口"，一期不做真实对接。

### 3.4 数据核对

**目标**：跨模块一致性校验，通过后才能生成申报数据。

- **质量评分**：如 82%，"10 项规则已通过 8 项"+ 进度条。
- **核对清单项**（示例规则）：
  1. 员工工资与个税匹配（来源：工资表、个税测算）
  2. 销项发票与银行回款匹配（来源：发票、银行流水）
  3. 进项发票认证状态（3 张待认证 → warn）
  4. 无票支出说明完整性（待补充 1 条 → warn）
  5. 企业所得税成本构成（工资/费用/无票支出）
- 每项状态：通过（绿）/ 待处理（橙），提供"处理"动作打开详情。
- **AI 核对建议（PC 时间线）**：优先级排序建议（先进项认证 → 未匹配流水 → 无票支出 → 生成申报）。
- 通过后操作："生成申报数据" → 跳转申报中心。

### 3.5 申报中心

#### 3.5.1 申报批次信息

- 当前申报期、纳税人类型、生成时间、状态（待提交）。
- 顶部标签：数据核对通过度（如 82%）。
- 批次基于已确认的工资、发票、银行流水、无票支出等真实数据生成；数据发生变更后可重新生成。

#### 3.5.2 三大税种测算卡（真实数据 + 公式 + 跳转验证）

| 税种 | 计算逻辑 | 关键数据汇总 | 跳转验证 |
| --- | --- | --- | --- |
| **增值税** | 应纳增值税 = 当期销项税额 − 当期可抵扣进项税额 + 附加税预估（城建税 + 教育费附加 + 地方教育附加，按增值税额 × 适用费率）。小规模纳税人按征收率计算。 | 销项总额 / 销项张数、进项总额 / 进项张数、可抵扣税额、待认证张数、附加税预估、应纳增值税 | 点击“查看销项”跳转发票管理（销项筛选）；点击“查看进项”跳转发票管理（进项/待认证筛选） |
| **企业所得税** | 应纳税所得额 = 营业收入 − 营业成本 − 税金及附加 − 期间费用（含工资成本、费用报销、无票支出等）；应纳所得税 = 应纳税所得额 × 税率（一般 25%，小微企业按优惠政策）。 | 营业收入、营业成本、工资成本、费用成本、无票支出、利润总额、适用税率、应纳所得税 | 点击“查看收入”跳转发票管理（销项+未开票收入）；点击“查看成本费用”跳转工资/发票/无票支出；点击“查看工资成本”跳转员工工资 |
| **个人所得税** | 工资薪金个税：对每位员工 `应税收入 = 应发工资 − 个人社保 − 个人公积金 − 专项扣除 − 5000`，按 7 级超额累进税率计算，汇总得当期应扣缴个税。 | 员工人数、已确认人数、工资总额、个人社保公积金合计、专项扣除合计、应税收入合计、应扣缴个税 | 点击“查看工资明细”跳转员工工资 |

- 每个税种卡片展示：应缴税额、计算过程关键项、状态、数据汇总（如销项总额/进项总额/收入总额/成本总额/员工人数）。
- 提供“重新生成”按钮，根据最新数据重新测算。
- 注：一期“个人所得税”统一为**工资薪金个人所得税扣缴申报**，后续按所得类型细分。详见 §6 待澄清。

#### 3.5.3 申报表预览

- 点击"前往申报"打开对应税种申报表（VAT / CIT / PIT）。
- 一期以**申报表截图**形式横向/纵向滑动预览（图片在 `tax-forms/` 目录）：
  - 增值税及附加税费申报表（一般纳税人适用）
  - 企业所得税月（季）度预缴纳税申报表（A 类 · 2025 版）
  - 个人所得税扣缴申报表（利息、股息、红利所得适用）
- 一期**不做真实税务局提交**，仅展示流程；"下载申报表"为演示入口。

## 4. 业务流程

### 4.1 主流程（三步报税 → 申报）

```
登录 → 选择企业 → 报税工作台
   │
   ├─ [满足条件] 开启零申报 ──► 自动完成（仅需确认）
   │
   ▼
Step1 员工工资（录入/导入 → 社保公积金 → 个税测算 → 确认本月工资）
   ▼
Step2 发票采集（销项/进项自动采集 + 费用小票/区块链/无票支出手动上传）
   ▼
Step3 银行流水（添加账户 → 上传/直连 → 匹配发票/工资/费用）
   ▼
数据核对（规则校验 + AI 建议 → 处理异常 → 通过）
   ▼
申报中心（三大税种测算 → 预览申报表 → [一期止于预览]）
   ▼
报表与凭证归档（三表 + 凭证 + 本期总结）
```

### 4.2 关键分支

- **零申报分支**：满足四条件 → 风险二次确认 → 开启；若后续产生开票/付款需提示撤销。
- **异常处理分支**：异常发票 / 未匹配流水 / 无票支出缺失 → 进入数据核对待办 → 处理后重新核对。
- **多企业分支**：切换企业 → 加载对应账套与申报期上下文。
