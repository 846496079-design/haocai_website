# 1\.页面整体布局（Mobile Layout）



所有移动端 AI 产品统一采用 **顶部导航 \+ 内容区域 \+ 底部导航** 的布局方式。



页面整体结构如下：



```Plain Text
┌──────────────────────────────┐
│     系统状态栏 Status Bar      │
│  时间 / 信号 / 电量 / 灵动岛    │
├──────────────────────────────┤
│      顶部企业信息栏 Topbar      │
│  企业名称 / 纳税类型 / 申报期   │
│  右侧：通知 / 快捷入口          │
├──────────────────────────────┤
│      页面标题区 Page Header     │
│  页面标题 / 页面说明 / 操作按钮 │
├──────────────────────────────┤
│                              │
│                              │
│      主内容滚动区 Content      │
│                              │
│  Card / List / Form / AI Chat │
│                              │
│                              │
├──────────────────────────────┤
│       底部标签导航 Tab Bar      │
│ 工作台 / 三步 / 申报 / AI / 我的│
├──────────────────────────────┤
│      iOS Home Indicator       │
└──────────────────────────────┘
```





## 页面结构说明



|区域|高度|说明|
|---|---|---|
|状态栏|系统默认|iOS / Android 系统状态栏|
|顶部导航栏|56px|页面导航、返回、标题、操作按钮|
|页面标题|56\~72px|页面名称、描述、筛选条件|
|主内容区域|自适应|页面主体内容，可滚动|
|底部导航栏|80px|一级导航入口（Tab Bar）|



> 页面滚动时，仅 **主内容区域** 允许滚动，顶部导航与底部导航保持固定。
> 
>

# 8\. Design Rules（开发规范）



所有 AI 产品统一遵循：



✅ 使用统一 Design Tokens



✅ 使用统一 Typography



✅ 使用统一 Radius



✅ 使用统一 Shadow



✅ 使用统一 Component Style



禁止：



❌ 新增品牌颜色



❌ 修改字体体系



❌ 修改圆角规范



❌ 修改基础间距



❌ 每个项目重新设计视觉风格



> 所有移动端 AI 产品（AI 财税 App、AI Agent App、AI 工作台等）均基于本 Design Tokens 实现，确保设计、研发与 AI 生成工具保持一致。
