# CMS 管理员初始化

## 当前账号模型

第一期只有一个独立 CMS 管理员，不复用财税产品账号。密码以 bcrypt 哈希存入 `cms_admin`，会话只存 SHA-256 token 哈希。初始化逻辑仅在 `cms_admin` 为空时使用环境变量创建账号。

## 首次创建

1. 确认 Vercel 环境已连接正确的 Neon 数据库并完成 schema 迁移。
2. 在对应 Vercel Environment 设置 `CMS_ADMIN_USERNAME` 和随机 `CMS_ADMIN_PASSWORD`。密码少于 12 位会被拒绝。
3. 重新部署该环境。
4. 访问该环境的 `/cms/login/`，使用初始凭据登录。首次登录请求会在空表中创建管理员。
5. 登录后访问 `/api/cms/health/`，确认数据库与 Blob 状态。
6. 查询 Neon 确认只有一条启用管理员记录，并确认 `password_hash` 以 bcrypt 格式存储；不要复制或展示哈希。
7. 将凭据保存到团队批准的密码管理器。确认登录正常后，从 Vercel 删除 `CMS_ADMIN_PASSWORD` 并重新部署，减少明文初始密码驻留时间。

移除初始密码不会影响已有管理员，因为后续认证读取数据库哈希。

## 锁定规则

连续 5 次错误密码会锁定 15 分钟。锁定期间的继续尝试不会提前解除锁定。不要通过直接清空数据库解决锁定；正常等待锁定到期并核对审计日志。

## 遗失密码或账号应急恢复

当前 CMS 尚无网页改密功能。需要重置时使用受控维护窗口：

1. 冻结 CMS 写入并确认操作者身份。
2. 创建 Neon 恢复点或备份。
3. 生成新的强密码并保存到密码管理器。
4. 在可信本机使用项目已锁定的 `bcryptjs` 生成 cost 12 哈希；密码只通过临时环境变量或安全提示输入，不写进命令、SQL 文件和日志。
5. 使用参数化数据库客户端更新目标管理员的 `password_hash`，同时把 `failed_login_count` 设为 0、`locked_until` 设为 NULL、更新 `updated_at`。
6. 撤销该管理员所有未过期会话：将 `cms_session.revoked_at` 更新为当前时间。
7. 登录验证并记录审计/变更工单，立即清除本机临时秘密。

不要删除 `cms_admin` 后依赖初始环境变量“重建”，这会破坏管理员 ID 与历史审计的关联。

## 凭据泄露

若初始密码、会话 Cookie 或数据库连接串出现在截图、聊天或仓库历史中：立即撤销相关会话，轮换密码和连接凭据，检查 `cms_audit_log` 与 Vercel/Neon 访问记录；仅删除当前文件不足以完成处置。
