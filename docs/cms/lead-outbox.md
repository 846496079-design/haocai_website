# 官网线索可靠中转运维说明

## 1. 工作方式

公开表单先写入官网服务器的 `.data/lead-outbox.sqlite`，写入成功后立即向用户返回已接收。PM2 `zds-lead-worker` 每 15 秒调用本机内部接口，由网站进程领取到期任务并转发到外部线索池。

外部接口不可达、返回 5xx、429 或超时时，任务按递增间隔重试；明确的非临时 4xx 会暂停任务，等待管理员检查数据并人工重试。只有外部接口明确返回 2xx 才标记为已发送。

## 2. 后台操作

登录 `/cms/` 后进入“线索管理”：

- 查看今日接收、待发送、超过 6 小时、已发送和已暂停数量。
- 使用“全部线索｜用户线索｜代理商线索”三个页签区分业务类型，并按状态和超时状态筛选。
- 页签数量表示各类型线索总量；顶部运行状态统计继续表示全站线索发送健康状态。
- 查看脱敏列表和完整详情、每次发送结果及耗时。
- 对未送达线索执行“立即重试”或“暂停”。

超过 6 小时仍未送达时，官网后台显示红色告警。配置飞书群机器人后，首次超时及持续故障每 6 小时最多发送一次汇总，全部恢复后发送一次恢复通知。

当前生产通知通道仍是 `LEAD_ALERT_FEISHU_WEBHOOK_URL`。企业自建应用机器人、DeepSeek 智能问数和应用机器人预警已完成设计与配置基础，但飞书长连接 SDK worker 尚未启用；完成真实 SDK、权限和会话验收前不得移除 Webhook。

## 3. 故障判断

1. 后台有待发送线索但尝试次数持续增加：外部接口仍异常，查看最近 HTTP 状态和脱敏错误。
2. 待发送线索存在但尝试次数不变化：检查 PM2 `zds-lead-worker` 和 `zds-website` 是否在线。
3. 后台提示飞书未配置：设置 `LEAD_ALERT_FEISHU_WEBHOOK_URL` 后重启两个进程。
4. 后台无法打开线索数据：检查共享 `.data`、数据库权限和加密密钥，不要生成新密钥覆盖旧值。
5. 智能问数配置校验失败：检查飞书允许名单、告警群是否位于允许群，以及 DeepSeek 地址是否为 HTTPS；该问题不影响线索接收与转发。

## 4. 数据与隐私

姓名、电话、公司和备注使用 AES-256-GCM 加密落盘；列表默认隐藏电话号码中段。数据库、备份和日志都不得包含密钥，飞书通知不得包含线索明文。

当前线索和发送记录永久保存。任何清理、归档或加密密钥轮换都必须另行制定迁移方案，不得直接删除数据库行或替换密钥。
