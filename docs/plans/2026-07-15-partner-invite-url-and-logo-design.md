# 代理邀请码 URL 与官网 Logo 更新设计

> 2026-07-20 更新：本文中的 `/cn/?c=邀请码` 首页跳转规则已由 `docs/superpowers/specs/2026-07-20-direct-product-entry-pricing-cycle-invite-design.md` 替代。当前仅 `/cn/partners/` 页面识别并预填邀请码；Logo 更新改由 `docs/superpowers/specs/2026-07-20-current-folder-logo-refresh-design.md` 维护。

## 1. 背景与目标

官网需要支持投放链接 `https://zhangdashi.ai/cn/?c=LFFWTL`。访问该链接时，用户应直接进入中国站代理合作意向表单，邀请码栏自动填写 `LFFWTL`，并允许用户修改后提交。

同时，官网中使用的品牌 Logo 需要同步为公司介绍资料目录中的最新中文、日文和繁体中文横版文件。

## 2. 已确认方案

### 2.1 URL 跳转与参数预填

- 中国站首页读取查询参数 `c`，同时兼容大写 `C`。
- 参数去除首尾空格后仍有值时，由服务端跳转到 `/cn/partners/?c=<编码后的值>#partner-form`。
- 空参数不触发跳转，`/cn/` 保持原有首页行为。
- 代理合作页继续支持原有 `inviteCode` 参数，并新增 `c`、`C` 别名；读取优先级为 `inviteCode`、`c`、`C`。
- 参数仅作为邀请码输入框的初始值，不直接写入 HTML，也不自动提交。
- 邀请码按代理线索池接口要求去除首尾空格并截取前 50 个字符，用户仍可编辑。
- 为表单区域增加稳定锚点 `partner-form`，跳转后直接定位到填表区域。

### 2.2 Logo 更新

- 中文站使用 `公司介绍/好财Logo/中文横版.png`。
- 日本站使用 `公司介绍/好财Logo/日本横版.png`。
- 香港站使用 `公司介绍/好财Logo/繁体中文横版.png`。
- 将上述最新源文件生成到带日期版本号的轻量网页 Logo 路径，导航、页脚和公司介绍页统一引用新路径，避免 Next.js 图片优化缓存继续返回旧 Logo。
- 原始大尺寸素材继续保留在公司介绍资料目录，不改变源文件内容和目录结构。

## 3. 未采用方案

- 客户端加载后再跳转：会先显示首页再跳转，存在闪屏，投放体验较差。
- 在首页弹出或嵌入表单：不符合“直接进入代理合作填表页面”的明确要求。
- 新增一套 Logo 内容配置字段：会扩大内容模型改动范围；当前只需要更新三个已有 Logo 引用点的资源路径。

## 4. 验收标准

- 访问 `/cn/?c=LFFWTL`，响应跳转到 `/cn/partners/?c=LFFWTL#partner-form`。
- 跳转后的代理合作表单邀请码栏显示 `LFFWTL`，且可以修改。
- 访问 `/cn/?C=LFFWTL` 同样生效。
- 访问 `/cn/` 或 `/cn/?c=` 不跳转。
- 直接访问 `/cn/partners/?inviteCode=LFFWTL` 的原有功能继续生效。
- 提交表单时，URL 参数仍能作为邀请码回传兜底值。
- 中文、日文、香港站的导航、页脚和公司介绍页展示各自最新横版 Logo。
- `npm run build` 通过，页面没有中文乱码。
