# CMS 图片上传生产容错实施计划

## 1. 固化图片处理边界

- 新增独立的图片文件头校验与处理模块。
- 动态加载 `sharp`，保留现有旋转、尺寸限制、透明图 PNG、普通图 JPEG 和缩略图逻辑。
- 将原生模块加载失败或已验证图片的转换失败降级为保存原图。
- 保持非法图片、权限、存储和数据库错误为失败。

## 2. 修复上传接口

- 上传路由移除顶层 `sharp` 导入。
- 根据处理模式保存两份优化资源或一份原图。
- 返回 `processingMode` 和可读提示，素材记录接口保持不变。

## 3. 修复 standalone 产物

- 在 Next.js 输出追踪中显式包含 `sharp` 与 `@img` 运行文件。
- 发布包生成阶段验证当前平台原生文件已进入最终 release。

## 4. 修复编辑器用户引导

- 安全解析 JSON、纯文本和空响应。
- 增加上传状态机，显示尚未上传、上传中、上传成功、已保存原图和上传失败。
- 原图降级给出可继续保存、预览和发布的下一步提示。

## 5. 验证与部署

- 新增图片处理、响应解析和界面状态专项验证。
- 执行 CMS 类型检查、现有翻译和发布引导回归、完整构建和发布包生成。
- 检查 standalone 原生文件和上传路由加载。
- 推送生产触发分支并验证线上路由不再在模块初始化阶段返回 500。
