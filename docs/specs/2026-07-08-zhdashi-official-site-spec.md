# 好财账大师官网 Spec

## 1. 目标

建设一个可上线的好财账大师宣传官网，用于承接投放、自然搜索、客户转介绍和销售沟通。首期以中国站为完整母版，日文站、香港站复用同一套组件和内容结构，只替换语言、地区化说明、联系方式、币种和合规提示，避免重复开发。

官网必须面向外部客户表达，不把“产品阶段、MVP、未接入、不承诺、真实登录入口”等内部交付控制点放在首屏、价值条、核心功能、定价卡等主要销售位置。涉及服务范围的内容统一使用“服务说明”“常见问题”“页脚提示”等客户可理解的表达。

官网核心转化目标：

- 引导用户进入产品试用登录页：`https://finance-ai.haocai360.cn/sign-in`。
- 引导高意向用户联系姚经理：`17511647049`、`shangjiacheng2@creya.cn`。
- 清晰说明 360 元/年、折合不到 1 元/天的单一定价，降低询价成本。
- 让企业主、财务、代账会计和被授权人能理解各自的使用路径。

## 2. 范围

### 2.1 首期开发范围

- 中国站完整落地页：`/cn/`。
- 统一营销页组件：`components/marketing/official-site.tsx`。
- 统一内容配置：`lib/site-content.ts`。
- 日文站、香港站复用同一组件：`/jp/`、`/hk/`，内容必须完成对应语言或繁体中文地区化，不在前台混用简体中文。
- 导航、页脚、SEO 文案同步调整为真实可上线口径。
- 本地可视化预览页面，支持后续共同评审。
- 首屏使用真实产品移动端截图作为产品展示资产。
- 直接试用路径拆分为“领取 7 天体验套餐”：先说明注册/登录、创建企业、领取体验、客服承接，再跳转产品登录页。
- 新增“代理合作”独立页面，面向分销商、代账机构、财税顾问、企业服务渠道收集合作信息。
- 首页新增企业现场图片信息流，用高端、简洁的图文方式展示团队、服务和客户沟通氛围。

### 2.2 不在首期范围

- 表单提交、CRM 入库、在线支付。
- 独立文章系统、帮助中心后台、客户案例详情页。
- 真实税局申报、银行直连、短信网关、OCR 识别等产品未接入能力的宣传承诺。
- 新增第三方统计、客服插件或地图 SDK。

## 3. 真实宣传边界

官网文案必须和 PRD、产品源码现状一致。

### 3.1 可以明确宣传

- 产品定位：面向中小企业和代账场景的 AI 智能做账报税工作台。
- 核心流程：员工工资、发票采集、银行流水、数据核对、生成申报。
- 已接入真实后端数据与接口，覆盖认证、企业、工资、发票、银行、核对、申报、报表凭证、AI、设置、帮助中心等模块。
- AI 财税助手已接入 DeepSeek，支持流式输出和上下文辅助。
- PC 后台与移动端均有产品形态。
- 单一定价：360 元/年。
- 联系信息和企业信息可展示。

### 3.2 必须谨慎表达

- “报税”统一表达为“辅助测算、生成申报数据、预览申报资料、流程化管理”，不写“自动向税局提交”。
- “AI”统一表达为“辅助核对、解释、总结、建议”，不写“完全替代会计”。
- “发票识别/OCR”不写“秒级识别”“准确率 99.7%”等无法证明表述。
- “银行直连”不写自动完成或强绑定外部机构的承诺，以资料归集、上传、匹配、核对流程表达。
- “准确率、安全认证、赔付承诺、客户数量”没有证据前不出现具体数字。

### 3.3 禁止出现

- 100% 准确、绝不出错、全税种罚款包赔。
- 已支持全国税局自动申报、RPA 直连税局。
- 等保三级、银行级认证、10 万企业、200 万企业等未提供证明的数据。
- 替代会计、无需财务人员、完全自动完成纳税申报。

### 3.4 外宣表达原则

- 可以大胆讲用户收益：更便宜、更省心、更实时、更规范、低至不到 1 元/天、手机随时查账、AI 帮你整理材料、风险提醒、专属客服。
- 可以参考伽玛账的内容结构：痛点提问、四个核心价值、软件核心能力、价格套餐、使用教程/流程、客服承接。
- 不照搬竞品数据和承诺：不使用“200 万企业”“98% 推荐”“1 分钟完成”“100% 杜绝错误”“错漏报全额赔付”等无法证明内容。
- 对“银行直连、税局提交、OCR”等敏感能力，在主页面用“发票/流水归集、申报材料生成、风险提醒、人工确认”替代表达，具体服务说明只在 FAQ 中说明。

## 4. 用户与转化流程

### 4.1 小微企业主

入口：搜索、朋友推荐、销售转发。

流程：

1. 看到首屏价值：360 元/年，用 AI 把月度做账报税流程标准化。
2. 阅读痛点：票据分散、不会算税、怕漏申报、看不懂报表。
3. 理解三步流程：工资、发票、流水先归集，再核对和生成申报资料。
4. 查看价格和客服。
5. 点击“开始试用”进入登录页，或联系姚经理。

### 4.2 企业财务/出纳

入口：老板转发、内部评估。

流程：

1. 查看功能模块：工资、发票、银行流水、数据核对、申报测算、报表凭证。
2. 关注数据可追溯：每个税额能回到工资、发票、流水来源。
3. 关注异常待办：待认证发票、待匹配流水、无票支出说明。
4. 进入试用，验证是否适合现有流程。

### 4.3 代账会计

入口：代账机构负责人或会计自己评估。

流程：

1. 看到多企业管理、授权、待办集中处理。
2. 了解同一流程服务多套企业账。
3. 关注 AI 辅助核对和申报前检查。
4. 试用后导入或录入样例企业，验证批量处理效率。

### 4.4 企业老板/经营者

入口：移动端访问。

流程：

1. 看到移动端可查报税进度、预计税额、待办和报表摘要。
2. 明白自己不需要每天进后台，只需要看关键结果和待办提醒。
3. 联系客服或进入登录页。

### 4.5 被授权人

入口：企业授权后访问。

流程：

1. 了解一个账号可管理或被授权操作多个企业。
2. 看到企业切换、权限协同和多角色配合。
3. 进入登录页完成操作。

## 5. 页面信息架构

中国站页面顺序：

1. 导航栏：品牌、产品介绍、公司介绍、代理合作意向、站点、开始试用；顶部导航必须作为分页导航使用，不作为页内锚点导航。
2. 首屏：主标题、低价心智、两组 CTA、产品价值展示。
3. 价值条：更便宜、更实时、更省心、更规范。
4. 产品首页 `/cn/`：左侧悬浮页内产品目录，右侧承载痛点、案例、核心功能、适用角色、工作流程、定价、服务说明和 FAQ。
5. 公司介绍页 `/cn/company/`：仅选用公司资料中与财税、账大师、AI 财税系统、客户服务经验相关的内容，不展示上海落户、法律、园区、金融等非财税业务。页面需要体现专业性，使用企业现场图片、服务能力、客户规模、团队交付方法组织内容。
6. 代理合作意向页 `/cn/partners/`：展示合作价值，表单收集姓名、电话、城市、公司、角色、合作方式、客户规模和补充说明。
7. 联系与页脚：企业主体、统一社会信用代码、地址、总部、电话、邮箱。

## 5.1 转化流程

### 直接试用

用户点击首屏、定价或最终 CTA 的“立即免费试用”后，不直接跳走，而是先弹出“领取 7 天体验套餐”说明。

流程：

1. 用户确认需要手机号注册或登录。
2. 注册/登录后创建企业账套，用于保存试用数据。
3. 进入产品后按 `trial=7d` 参数识别 7 天体验套餐来源。
4. 用户在产品内跑通工资、发票、流水、核对、报表流程。

试用弹窗内不展示“先电话咨询”按钮，避免分散注册试用路径。

### 电话咨询

页脚和代理合作成功弹窗可展示姚经理联系方式。底部最终 CTA 和试用弹窗不展示电话咨询组件。

### 代理合作意向

代理合作意向表单采用代理线索池资料提交接口完成线索回传：`POST https://hcagent.ai-hc.cn/api/v1/agent-public-pool-leads/submit`。前端校验必填项后，以 JSON 方式提交 `contactName`、`contactPhone`、`companyName`、`position`、`city`、`cooperationMode`、`customerScale`、可选 `inviteCode` 和 `remark`。必填项只包括姓名、电话、城市；公司/团队、角色身份、合作方式、客户资源规模、邀请码/邀请人电话和补充说明均为选填。字段映射为：姓名对应 `contactName`，电话对应 `contactPhone`，城市对应 `city`，公司/团队对应 `companyName`，角色身份对应 `position`，客户资源规模对应 `customerScale`，合作方式对应 `cooperationMode`，邀请码/邀请人电话对应 `inviteCode`，补充说明对应 `remark`。`companyName`、`position`、`city`、`cooperationMode`、`customerScale`、`inviteCode` 按接口要求限制在 50 个字符以内。若 URL 中存在 `inviteCode` 参数，表单自动预填，用户可修改。必填项未填写时，在对应字段下方展示红字“该项为必填项”，不触发浏览器默认“请填写此字段”提示。提交成功后页面弹出姚经理电话和企业微信二维码，弹窗使用“合作意向已收到”作为标题；接口返回 IP 超限或失败提示时，在表单底部展示接口提示。

## 6. 视觉与交互规范

- 采用现有 UI 规范的轻科技、可信、企业级 SaaS 风格。
- 保持品牌色 `#5B6CFF`、渐变 `#4F7BFF` 到 `#6C5CFF`。
- 字体使用 Inter、PingFang SC、Microsoft YaHei 兜底。
- 官网顶部横向导航只放分页入口；产品介绍内部的核心功能、角色、流程、定价、FAQ 等页内目录放在大屏左侧悬浮目录中。悬浮目录采用简洁、轻量、半透明样式，不遮挡正文，小屏隐藏。
- 不使用夸张营销视觉，不使用暗黑重科技风格。
- 产品试用 CTA 指向登录页；代理合作表单通过线索池 API 提交；页脚保留 `tel:`、`mailto:` 联系方式。
- 移动端导航折叠，正文先显示价值、价格、试用和联系方式。

## 7. 技术设计

### 7.1 组件结构

- `lib/site-content.ts`：集中维护各站点内容、导航、CTA、FAQ、价格、企业信息。
- `components/marketing/official-site.tsx`：统一渲染官网页面。
- `components/layout/Navbar.tsx`：改为接收站点配置，不写死夸大文案。
- `components/layout/Footer.tsx`：改为接收站点配置，展示真实企业信息。
- `app/cn/page.tsx`、`app/jp/page.tsx`、`app/hk/page.tsx`：只选择对应内容并渲染。

### 7.2 多语言策略

- 中国站是母版内容。
- 日文站、香港站使用同一数据字段，但前台展示内容必须完成本地化；公司介绍、代理合作表单、成功弹窗、功能卡片、角色卡片不得残留简体中文母版文案。
- 后续新增地区只需要增加内容对象，不新增页面组件。
- 所有站点共享 CTA 链接，默认进入同一个登录页。
- 以后所有官网更新默认同步检查中国站、日本站、香港站三个主站；除非需求明确只改某一个地区站，否则组件、内容字段、交互状态和验收都要覆盖三站。
