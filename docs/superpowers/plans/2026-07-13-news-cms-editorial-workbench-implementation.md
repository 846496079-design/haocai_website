# 新闻 CMS 稿件工作台实施计划

## 第一阶段：领域模型与本地迁移

1. 扩展状态为草稿、已发布、已下架、回收站；为文章补充分类、删除时间、置顶与排序字段。
2. 新增分类、内容块、素材、翻译任务与导入幂等表；编写 SQLite 开发迁移，保留现有新闻和管理员数据。
3. 将中文正文作为源稿，建立日文、繁体中文翻译状态；保留现有三语内容，避免数据丢失。

## 第二阶段：CMS 工作台与编辑器

1. 重构新闻列表为四个状态工作区，增加关键词、日期、分类、标签、置顶筛选与新建入口。
2. 新增分类管理页面与 API，支持新建、改名、停用、合并与引用保护。
3. 将编辑器改为日期选择、分类选择、内容块编辑、正文图片上传、拖拽排序与草稿保存。
4. 在已发布工作区实现置顶和手动排序；其他状态不显示排序动作。

## 第三阶段：发布、翻译与导入

1. 实现删除、恢复、永久删除、下架与发布事务，并写入审计记录。
2. 实现翻译供应商适配器与一键翻译任务；未配置凭据时返回明确配置错误，不伪造译文。
3. 预留受签名保护的工作台导入接口，支持未知分类自动创建、缺失分类提示和幂等处理。

## 第四阶段：生产存储与部署

1. 增加 Neon Postgres 存储实现与迁移脚本，生产环境由 `DATABASE_URL` 切换。
2. 增加 Vercel Blob 素材存储实现，生产环境由 `BLOB_READ_WRITE_TOKEN` 启用。
3. 更新环境变量模板、部署说明、迁移与回滚步骤。

## 验证顺序

1. 迁移现有 SQLite 数据并验证管理员、历史新闻、公开页不回归。
2. 覆盖草稿、发布、下架、删除、恢复、分类合并、排序和翻译失败场景。
3. 运行 `npm run build`，在桌面与移动端检查 CMS 工作台、文章编辑和公开新闻页。
4. 配置 Vercel、Neon、Blob 与翻译环境变量后，完成线上预览部署和生产发布验证。
