# CMS 统一富文本与微信公众号兼容排版实施计划

## 目标

把 CMS 正文从 `lead / sections / closing` 迁移为单一结构化富文本模型，提供所见即所得编辑、确定性内联 HTML、官网统一渲染和微信公众号富文本复制，并保持现有三语、草稿、预览、发布、下架和审计链路。

## 实施步骤

### 1. 建立内容契约与渲染器

- 在 `lib/cms/` 增加富文本节点、样式配置、素材引用和发布快照类型。
- 增加旧新闻到新模型的幂等转换器。
- 增加节点白名单校验、HTML 转义、URL 校验、模板映射、内联 HTML 渲染和 SHA-256 哈希。
- 让 SQLite 与 Postgres 的 `content_json` 读取时自动规范为新模型，写入时只写新模型。
- 增加迁移验证脚本，比较正文文字和图片顺序。

### 2. 接入所见即所得编辑器

- 使用 MIT 协议的 Tiptap 3.28.0，并显式关闭 SSR 立即渲染。
- 编辑器依赖只从 CMS 客户端组件加载。
- 支持标题、段落、加粗、斜体、下划线、删除线、颜色、背景色、字体、字号、对齐、列表、引用、链接、分隔线、代码、图片和表格。
- 支持图片选择、拖放和剪贴板粘贴，统一调用现有 CMS 上传接口。
- 提供模板和排版参数、手机宽度实时预览、保存状态和错误状态。

### 3. 替换 CMS 正文编辑与翻译

- 保留基础信息、三语切换、翻译和发布检查。
- 删除按分节逐项编辑的运行界面，替换为统一富文本编辑器。
- 更新翻译服务，只翻译结构化文档中的文字节点，保留节点和样式。
- 保存前由服务端重新渲染并忽略客户端提交的发布 HTML。
- 复制时使用已保存快照，同时写入 `text/html` 和 `text/plain`，记录审计日志。

### 4. 切换官网预览与公开渲染

- 官网新闻详情直接渲染已净化的 `publicationHtml`。
- 列表仍使用独立标题、摘要、日期、分类、标签和封面字段。
- 预览与正式页面复用相同的正文组件。
- 公共页面不导入 Tiptap 或其他编辑器运行时代码。

### 5. 迁移、文档与验证

- 增加 SQLite/Postgres 幂等迁移命令和迁移报告。
- 更新 CMS README、接口字段、数据操作和部署说明。
- 运行 CMS 类型检查、应用构建、渲染器验证、迁移验证和 UTF-8 检查。
- 启动本地 Next.js，验证 CMS 登录、稿件编辑、预览、公开 CN/JP/HK 页面。
- 使用 Chrome/Edge 验证富文本剪贴板；真实微信公众号粘贴作为人工终验。

## 完成定义

- CMS 新增和保存的正文只使用统一富文本模型。
- 旧内容可无损转换，迁移脚本可重复运行。
- 官网预览、官网正式正文和公众号剪贴板来自同一发布快照。
- 图片、文字层级、颜色、字号、字体白名单和布局均以内联样式输出。
- 原草稿、翻译、预览、发布和下架链路通过回归。
- 未修改用户现有的无关工作区文件。
