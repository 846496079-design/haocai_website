# 飞书 DeepSeek 官网线索智能问数机器人实施计划

## 1. 依据与范围

- 设计文档：`docs/superpowers/specs/2026-07-18-feishu-deepseek-lead-query-bot-design.md`
- 当前分支：`codex/official-home-trust-redesign`
- 本次先完成不含真实密钥的配置、查询计划契约、DeepSeek 兼容适配器和可独立验证的统计基础。
- 飞书官方 SDK 长连接 worker 在用户提供 SDK 或确认依赖版本后接入；未完成前不修改生产 PM2 启动项，避免发布不可运行进程。
- 保留现有官网、新闻 CMS、线索接收、线索转发和 Webhook 预警逻辑。

## 2. 实施步骤

### 步骤一：服务端配置契约

1. 在 `.env.example` 增加飞书应用机器人和 DeepSeek 变量，使用官方 DeepSeek HTTPS 地址和当前模型作为无密钥示例。
2. 新增 `lib/feishu/config.ts`，集中读取、规范化和验证配置。
3. 配置读取采用惰性调用；网站构建、官网访问和线索 worker 不因机器人变量缺失而失败。
4. App Secret 和 API Key 不进入日志、错误消息、客户端代码或版本控制。

### 步骤二：智能问数查询计划契约

1. 新增 `lib/feishu/query-plan.ts`，定义 `SUMMARY`、`COMPARE`、`TREND`、`PEAK` 和 `HELP` 查询计划。
2. 使用显式字段校验，拒绝未知枚举、未知字段、SQL 文本、个人明细请求和超出限制的时间范围。
3. 服务端按 `Asia/Shanghai` 计算预设、相对和明确日期范围，统一转换为 ISO 边界。
4. 只开放线索数量、类型、投递状态、超时状态、按日或按周分组。

### 步骤三：DeepSeek 兼容适配器

1. 新增 `lib/feishu/deepseek.ts`，使用标准 `fetch` 调用 OpenAI 兼容的 `/chat/completions`。
2. 请求启用 JSON Output，系统提示明确要求 JSON；设置非流式、超时、输出长度上限和低随机性。
3. 仅发送用户问题、当前北京时间和允许的查询能力，不发送数据库记录。
4. 对 HTTP 错误、超时、限流、截断、空结果和非法 JSON 返回稳定的脱敏错误类型。
5. 对适配器保留明确接口，用户后续提供 SDK 后可以只替换传输实现，不改变查询计划和统计模块。

### 步骤四：线索统计基础

1. 新增 `lib/leads/analytics.ts`，把已验证的查询计划转换为参数化 SQLite 聚合查询。
2. 查询只读取 `kind`、`status`、`created_at`、`delivered_at`，不读取或解密线索正文。
3. 支持合计、按类型、按状态、日/周趋势、时间段对比和峰值。
4. 对比基期为零时只返回差值，不计算无意义百分比。

### 步骤五：事件去重存储边界

1. 在现有线索 SQLite 初始化中增加 `feishu_bot_event` 表和唯一事件 ID。
2. 只保存事件 ID、脱敏用户/会话摘要、处理状态、意图、错误码和时间。
3. 提供领取、完成和失败接口，为后续长连接 worker 保证幂等处理。

### 步骤六：文档和 SDD 同步

1. 更新 CMS 环境变量清单和自托管运维手册。
2. 更新官网主 SDD，记录机器人、DeepSeek 和受控统计边界。
3. 明确当前状态：配置和问数基础已具备，飞书 SDK 长连接与真实生产凭据等待后续接入。

### 步骤七：验证

1. 增加无外网测试，验证配置校验、查询计划校验、北京时间边界和参数化统计。
2. 使用 mock `fetch` 验证 DeepSeek 请求不包含测试姓名、电话或数据库记录。
3. 验证超时、限流、非法 JSON 和截断响应不会触发数据库查询。
4. 执行 TypeScript 检查、线索现有测试、生产构建、中文乱码检查和 `git diff --check`。
5. 只提交本功能相关文件，保留工作区既有无关修改。

## 3. 后续 SDK 接入步骤

用户提供飞书 SDK 或确认官方 Node SDK 版本后：

1. 安装并锁定 SDK 依赖。
2. 新增 `scripts/feishu/worker.mjs` 和事件处理器。
3. 接入允许名单、事件去重、DeepSeek 查询计划、统计执行和飞书卡片回复。
4. 把应用机器人接入主动预警，保留 Webhook 单通道回退。
5. 更新发布打包、PM2 启动、健康检查和回滚逻辑。
6. 使用真实飞书自建应用完成单聊、群聊、问数和预警验收。

## 4. 本阶段验收标准

- 示例环境文件完整列出飞书和 DeepSeek 变量且不含真实凭据。
- DeepSeek 适配器可在 mock 环境生成并验证受控查询计划。
- 查询计划不能包含 SQL、个人明细或未知数据库字段。
- 统计模块结果与直接测试夹具一致，且不解密线索正文。
- 机器人配置缺失不影响网站构建、官网访问或线索 worker。
- SDD 和运维文档明确区分“配置基础完成”与“飞书 SDK 生产接入完成”。
