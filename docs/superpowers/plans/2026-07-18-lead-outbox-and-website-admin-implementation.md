# 官网线索可靠中转与官网后台实施计划

## 1. 依据

- 设计文档：`docs/superpowers/specs/2026-07-18-lead-outbox-and-website-admin-design.md`
- 当前分支：`codex/official-home-trust-redesign`
- 约束：保留现有新闻 CMS、公开页面和原子发布逻辑；只修改线索提交、官网后台入口及相关部署验证。

## 2. 实施步骤

### 步骤一：线索领域与本地持久化

新增 `lib/leads/`，实现：

1. 公开字段类型、状态、统计和后台视图类型。
2. 两类表单的服务端校验与规范化。
3. AES-256-GCM 加解密和稳定摘要。
4. 独立 SQLite schema、WAL、busy timeout、事务、重复提交抑制和频率控制。
5. 任务领取租约、成功/失败记录、递增重试、人工重试/暂停和每日在线备份。

### 步骤二：同域公开接口与前端切换

新增：

- `app/api/public/leads/trial/route.ts`
- `app/api/public/leads/partner/route.ts`

修改 `components/marketing/official-site.tsx`，把外部 URL 改为同域接口。只有官网本地持久化失败时显示失败；HTTP 202 按原有成功交互处理。

### 步骤三：异步转发、预警和内部调度

新增外部发送与飞书汇总通知逻辑，以及受 `LEAD_WORKER_TOKEN` 保护的内部处理接口。新增轻量 PM2 worker，每隔 15 秒调用本机接口；单批处理有上限，避免长时间占用。

外部请求统一设置超时、幂等请求头和脱敏错误记录。超过 6 小时首次预警，此后每 6 小时最多一次，恢复后通知一次。

### 步骤四：官网后台双板块

1. `/cms` 改为官网后台首页，展示“线索管理”和“新闻内容管理”。
2. 新增 `/cms/news/`，承载现有新闻管理列表；现有新闻编辑、分类、设置路径保持兼容。
3. 新增 `/cms/leads/` 与 `/cms/leads/[id]/`，实现统计、筛选、预警、发送历史、立即重试和暂停。
4. 新增受现有管理员会话保护的线索管理 API。

### 步骤五：部署与配置

1. 发布打包时包含 worker 文件。
2. 发布脚本在共享环境中首次生成线索加密密钥和 worker token，不覆盖已有值。
3. 同时管理 `zds-website` 与 `zds-lead-worker`，发布失败时一起回滚。
4. 更新环境变量、部署手册和数据操作文档。
5. 飞书 Webhook 未配置时后台明确显示，线索接收和转发仍正常。

### 步骤六：验证

1. 使用临时 SQLite 和测试密钥验证加密落盘、重复抑制、租约、重试、预警与恢复。
2. 执行 TypeScript 检查和生产构建。
3. 启动本地生产服务，验证未登录保护、公开提交、后台列表与内部 worker 鉴权。
4. 检查中文乱码、`git diff --check` 和发布包内容。
5. 只提交本次相关文件，保留仓库中已有的无关修改。

## 3. 验收标准

- 外部接口不可达时，两类表单均能在本地保存并向用户显示成功。
- 待发送任务在外部恢复后自动送达，发送历史可审计。
- 发布或 PM2 重启不丢失任务。
- 超过 6 小时在后台标红，并在配置 Webhook 后发送无个人信息的飞书预警。
- 官网后台清晰分为线索管理和新闻内容管理；原新闻管理功能不回归。
- SQLite 文件中检索不到测试姓名、电话和备注明文。
