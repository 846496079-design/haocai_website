# CMS 必填字段与发布校验引导实施计划

## 目标

让编辑器明确标识中文发布必填项，并让“确认发布”在非处理状态下始终可点击；点击后准确汇总全部缺项、显示字段错误并定位第一个待处理位置，同时保留保存、预览、审核和服务端发布门禁。

## 实施步骤

### 1. 统一必填字段定义

- 在 CMS 类型层定义稳定字段标识、中文名称和显示顺序。
- 新增用户可填写必填项检查函数，区分字段缺失与服务端发布快照缺失。
- 保留现有三语完整度和外语公开状态规则。

### 2. 增加结构化服务端校验

- 新增发布校验错误类型，携带错误码、缺项字段和可读消息。
- SQLite 与 PostgreSQL 发布函数共用相同中文必填检查。
- 发布接口将已知错误序列化为结构化响应，未知错误保持现有兜底。

### 3. 改造编辑器必填与定位交互

- 仅在简体中文页为分类、新闻标题、摘要、新闻封面和正文内容显示 `*`，并说明其余字段选填。
- 为缺项字段增加红色边框、字段错误、`aria-invalid` 和错误关联。
- 点击发布时先汇总中文缺项，再依次检查保存、预览和人工审核；切换到中文页并聚焦首个阻塞位置。
- 用户补全字段后即时清除对应错误；服务端结构化缺项可以反向更新前端错误。
- 发布按钮只在保存、上传、翻译或发布请求执行期间禁用。

### 4. 验证与文档同步

- 新增必填字段与发布引导专项验证，覆盖字段顺序、错误消息、按钮门禁和双存储边界。
- 更新既有翻译旅程验证，确保新按钮语义不会被旧断言误判。
- 同步 CMS 部署验收清单。
- 执行专项验证、非阻塞翻译运行时、CMS 类型检查、生产构建、发布包和公网验收。

## 回滚

- 本次不修改数据库结构或已发布数据，回滚前端、发布校验模块、接口响应和验证脚本即可恢复旧交互。
- 回滚不撤销非阻塞翻译发布、翻译锁定、版本预览和外语站过滤。
