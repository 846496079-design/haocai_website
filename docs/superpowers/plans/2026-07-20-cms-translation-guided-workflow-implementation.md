# CMS 翻译锁定与发布引导实施计划

## 目标

在保留人工预览和非阻塞翻译发布规则的前提下，锁定翻译过程中的整个编辑工作区，并让页面持续告诉用户当前阻塞原因和可执行的下一步操作。

## 实施步骤

### 1. 抽离编辑器旅程判定

- 新增无界面依赖的编辑器旅程判定函数，统一处理翻译、上传、保存、中文完整度、草稿状态、预览状态和人工审核的优先级。
- 返回单一当前动作，避免页面同时给出互相冲突的操作提示。

### 2. 增加翻译专用状态与页面锁定

- 在 CMS 新闻编辑器新增独立 `translating` 状态，不再使用 `saving` 代表模型翻译。
- 翻译开始后显示全屏遮罩，并通过 `inert` 和控件禁用状态阻止底层编辑与键盘操作。
- 翻译期间注册 `beforeunload` 提醒；程序自身在翻译成功后重新载入时允许正常离开。
- 为处理遮罩、成功引导和错误提示补充焦点管理及状态播报。

### 3. 串联翻译、预览和发布引导

- 翻译成功后使用当前标签页的会话状态记录成功结果，再重新读取服务端草稿。
- 在发布检查区增加持续的下一步引导卡，为保存和预览阶段提供直接操作入口。
- 预览完成后提示用户检查预览并返回勾选人工审核；人工审核未勾选时发布按钮保持禁用。
- 内容变化时重置人工审核，避免复用旧内容的审核确认。

### 4. 明确发布检查文案

- 中文不完整时显示“缺少必填内容”，不再把补全和保存合并成一个状态。
- 外语状态区分可随本次发布上线、译文有缺项、译文未就绪；外语状态不进入中文发布按钮门禁。
- 保留译稿新鲜度、预览和图片上传状态。

### 5. 自动验证与文档同步

- 新增旅程判定单元断言和编辑器静态边界检查，覆盖状态优先级、整页锁定、离页提醒、成功引导及非阻塞发布规则。
- 将专项验证加入 `package.json`，并在 CMS 部署验收文档补充真实操作旅程。
- 执行专项验证、现有非阻塞翻译验证、CMS 类型检查、生产构建、中文乱码扫描和 `git diff --check`。

## 回滚

- 本次不修改数据库或翻译接口契约，回滚前端编辑器、旅程判定文件和对应验证即可恢复旧交互。
- 回滚不会删除已生成译文、预览记录、审计记录或已发布版本。
- 非阻塞翻译发布是已确认的独立功能，不随本次交互回滚而撤销。
