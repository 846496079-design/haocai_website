# 公司页冗余内容与 H5 产品展示修正设计

## 问题

- 公司介绍页的“公司定位与服务方式”与后续“专业能力”内容表达重复；移动端即使压缩区块边距，仍会形成不必要的阅读旅程和视觉空白。
- 产品中心 H5 使用移动端截图，但官网额外叠加了一层设备卡片边框与内边距，缩小了实际产品画面；用户看到的是营销容器，而不是足够清晰的产品界面。
- 产品中心出现“解决方案不再单独拆页，统一放在产品中心”这一内部实现口径，不应出现在官网前台。

## 方案

### 公司介绍

- 删除“公司定位与服务方式”整个模块：包括移动端展开卡片和桌面端三张定位卡。
- 保留公司首屏统计卡、详细发展里程碑、专业能力与企业现场图片流。
- 删除后使统计卡片区直接衔接里程碑，移除只为该模块存在的移动端间距覆盖样式。

### H5 产品展示

- 桌面端维持当前带边框、阴影和浮层说明的产品展示卡片。
- 小于 `md` 的 H5 端不再使用官网外层设备框、边框和额外内边距；真实产品截图全宽展示，保留图片自身已有的设备视觉和圆角。
- H5 端图片继续按原始比例完整显示，不使用固定高度裁切；不伪造不可操作的产品交互，也不将截图描述为可直接登录的产品页面。

### 前台文案

- 将产品中心解决方案标题替换为面向客户的价值表达：
  - 中国站：`覆盖企业财税与经营协同的关键场景`
  - 日本站：`企業の財税と経営協同を支える主要シーン`
  - 香港站：`覆蓋企業財稅與經營協同的關鍵場景`
- 搜索并移除同类公开前台的开发过程、自我说明或搭建口径；不删除对用户有价值的产品边界说明。

## 影响范围

- `components/marketing/official-site.tsx`：删除公司定位模块，调整产品图片响应式容器。
- `lib/site-content.ts`：替换三站产品解决方案标题。
- `app/globals.css`：删除不再使用的公司定位移动端间距规则。
- `docs/specs/2026-07-08-zhdashi-official-site-spec.md`：同步页面结构和 H5 展示规范。

## 验收标准

- 公司介绍移动端的统计卡片后不再出现“查看公司定位与服务方式”，而是直接进入发展里程碑；没有对应的大块空白。
- 390px 宽度下，产品截图宽度接近页面可用宽度，完整显示原始内容；桌面展示卡片不变。
- 三站产品页不出现“解决方案不再单独拆页，统一放在产品中心”或同类内部搭建文案。
- `npm run build` 通过，cn/jp/hk 的公司页和产品页可访问且无横向溢出。
