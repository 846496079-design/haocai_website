# 公司介绍、产品展示与站点体验优化设计

## 目标

在不公开未审核法律文本、不夸大既有能力的前提下，提升官网页面切换、站点切换、公司介绍和产品中心 H5 展示体验；同时将公司里程碑内容固化为可维护的资料来源。

## 已确认事实与边界

- `公司介绍/用户协议（完整稿）.md` 与 `公司介绍/隐私政策（完整稿）.md` 均未完成法务审核。
- 两份稿件不得作为官网正式协议或隐私政策公开，不创建可访问的公开详情页，不在页脚保留无效入口。
- 协议和隐私政策继续保留在 `公司介绍` 文件夹，作为后续法务定稿和上线的源资料。
- 现有横向 Logo 图片约 1.6MB、原始宽度超过 10000px；顶部导航切页时重新挂载该图，造成重复解码和可见闪烁。
- 当前 H5 产品截图以固定高度和 `object-cover` 显示，裁切了实际界面内容。
- 公司介绍移动端的统计卡片区和“查看公司定位与服务方式”区相邻时，上下区块留白叠加，形成明显空白。

## 方案

### 1. Logo 资源与页面切换

- 保留现有原始 Logo 文件，不覆盖其作为源素材的用途。
- 生成适合网页显示的轻量横向 Logo 资源，并让顶部导航与页脚引用该资源；保留固定尺寸与 `priority` 策略，减少切页时的图片解码闪烁。
- 不改变品牌图形、颜色、导航结构或试用按钮行为。

### 2. 站点名称本地化

- 在站点选择菜单中，站点名称随当前站点语言显示：
  - 中国站：`中国站 / 日本站 / 香港站`
  - 日本站：`中国サイト / 日本サイト / 香港サイト`
  - 香港站：`中國站 / 日本站 / 香港站`
- 站点路径、代码标识和切换行为保持不变。

### 3. 公司介绍与里程碑

- 在公司介绍页的专业能力内容前新增“发展里程碑”模块，采用横向时间轴（桌面）与紧凑纵向时间轴（手机）。
- 里程碑扩展为 7 个阶段：`2014 起步`、`2016 场景洞察`、`2018 产品化沉淀`、`2020 数据协同`、`2022 智能化升级`、`2024 生态协作`、`2026 面向未来`。每个阶段均包含业务关注点、能力沉淀和客户价值，避免只展示年份和一句空泛口号。
- 除 `2014` 外，其余六个时间点按用户确认的官网阶段性叙事编写；不得混写为工商变更、融资、认证、客户数量或其他可核验事实。
- 在 `公司介绍/官网里程碑内容依据.md` 固化完整 7 阶段文案、官网阶段性叙事拟定稿标记、可替换年份位置、待校准事项和来源说明，作为后续修改依据。
- 调整移动端统计卡片区下方和定位展开卡片上方/下方的间距，消除连续 section padding 产生的空白；不删除公司介绍内容。

### 4. H5 产品真实界面展示

- 保持桌面端的产品展示卡片、边框、阴影和视觉层次。
- 手机端取消固定高度裁切，按图片真实比例完整展示产品界面；在有限宽度下通过更紧凑的卡片边距和合理最大宽度保持可读性。
- 不伪造桌面端产品截图，也不把移动端截图描述为桌面端功能。

### 5. 协议与隐私政策

- 本轮不公开未审核文件，不增加协议页路由。
- 页脚不展示无法访问的“用户协议”“隐私政策”文字入口，保留“关于我们”入口和现有企业联系方式。
- 法务定稿后，再补充正式版本号、生效日期、公开地址和页脚链接，并重新复核内容与实际数据流的一致性。

## 技术范围

- `components/layout/Navbar.tsx`：使用轻量 Logo，站点选择名称按当前语言显示。
- `components/layout/Footer.tsx`：使用轻量 Logo，移除未上线协议的无效入口。
- `components/marketing/official-site.tsx`：增加里程碑模块、调整公司页移动端区块间距、修复 H5 产品截图裁切。
- `lib/site-content.ts`：集中维护三站里程碑与站点名称本地化内容。
- `public/images/brand/`：新增轻量网页 Logo 资源，保留原图。
- `公司介绍/官网里程碑内容依据.md`：保存可维护的里程碑内容依据。
- `docs/specs/2026-07-08-zhdashi-official-site-spec.md`：同步正式规格。

## 验收标准

- 切换中国站内页面时，Logo 不再出现明显白闪或重新解码闪烁。
- 站点下拉菜单在 cn、jp、hk 三站分别显示对应语言的站点名称。
- 公司介绍页展示阶段性里程碑；手机端统计区与定位展开区域之间没有异常大空隙。
- 390px 手机宽度下，产品真实界面截图完整可见、无固定高度裁切；桌面端既有展示层次不退化。
- 页脚没有未审核协议/隐私政策的无效公开入口；未审核文件只留在资料目录。
- `npm run build` 通过，cn/jp/hk 的公司页和产品页可访问且无横向溢出。
