# 官网首页与内容重构设计

## 背景与目标

当前首页使用深蓝、蓝紫渐变与大面积光晕，视觉语言混杂；公众号区采用一张大卡片加两张纵向卡片，产生大量无效留白。首页还重复展示产品中心的亮点、痛点、流程和行业数据。客户案例页的“12 万+”与行业数量属于测算文案，不能以真实客户案例形式公开展示。新闻中心除首篇活动报道外，现有 SVG 配图风格不适合对外官网。

本轮目标是让首页成为简洁的品牌与转化入口，产品中心承担功能说明，案例页只呈现不带虚构规模的匿名反馈；同步提升新闻封面的编辑质感。

## 视觉与信息架构

### 视觉基调

- 使用深石墨黑、冷白和单一青蓝作为基础色与强调色。
- 禁用蓝紫渐变标题、强光晕、彩色漂浮球和大面积渐变填充。
- 保留克制的网格、边框、细微阴影与单色光线，表达科技感。
- 全站使用已放入 `public/images/brand` 的轻量横版 Logo：`zds-logo-h-*-web.png`。

### 首页

首页从“产品说明书”调整为以下顺序：

1. 品牌主视觉：一句核心价值、真实产品截图、试用与产品中心入口。
2. 使用入口：将公众号二维码、说明与试用按钮压缩成一张紧凑模块；移除无必要的空白和未确认的桌面端下载入口。
3. 价值摘要：仅保留三项非重复的经营价值摘要，并链接到产品中心；不重复功能列表、痛点卡片、工作流程或完整功能表。
4. 定价摘要：保留套餐与试用，完整功能对比按钮跳转至 `/product/#pricing`。
5. 客户反馈摘要：展示 2 至 3 条匿名反馈，并链接至客户案例页。
6. 新闻摘要与站内导航：保留。

### 客户案例

- 移除“12 万+ 企业”、十个行业数量、节省比例等无法证明的规模和效果数字。
- 使用“客户反馈”定位，卡片仅包含脱敏行业、角色、使用场景和反馈原话。
- 现有文字不标记为可核验客户名称；在获得公开授权素材前，一律采用匿名标签，不做客户数量或收益承诺。
- 首页复用同一组反馈数据的前 2 至 3 条，避免首页与案例页重复堆砌。

### 新闻

- 首篇 `2026-07-09` 保留现有真实活动图片作为封面。
- 其余文章按类别生成无文字、无水印、无人物肖像依赖的编辑风格封面，放入 `public/images/news/generated/`。
- 新闻详情页移除正文内的插图渲染；文章文本、日期、分类与首篇真实活动图片不改。

## 代码与数据边界

- `components/marketing/official-site.tsx`：调整首页、触达模块与案例渲染；保留已有试用表单和 API 提交逻辑。
- `lib/site-content.ts`：删除不可验证的行业规模数据，增加匿名反馈数据结构与中日港文案。
- `lib/news-content.ts`：仅替换封面路径；移除正文图片字段的消费，不重写文章正文。
- `app/*/cases`、品牌与新闻资源：保留现有多语言路由结构。
- 不处理工作区中与本轮无关的删除、未跟踪或改动文件。

## 验收标准

1. 首页不再渲染功能、痛点、流程及行业数量的重复模块。
2. 首页与案例页均不出现“12 万+”和行业规模测算数据。
3. 公众号模块在桌面与移动端均为紧凑布局，无大面积无内容空白。
4. 最新轻量 Logo 在 CN、JP、HK 导航与页脚正常显示。
5. 完整功能对比入口可定位至产品中心的 `#pricing`。
6. 除首篇真实活动报道外，新闻卡片均使用新的无文字封面；新闻详情不再显示段落插图。
7. `npm run build` 成功，且在 390px 和桌面宽度检查首页、产品中心、客户案例、新闻中心的布局与链接。

## 价格心智文案统一

- 全站面向客户的“小规模企业按天成本”统一使用“一天只要一块钱”。
- 不再展示精确日均价或“每天不到 1 元”等折算文案；价格仍保留 `360 元/年`，避免改变实际套餐价格。
- 日文与港文站使用当地可读表达，分别为“1日わずか1元”和“每天只要一元”，不保留小数日均价。
