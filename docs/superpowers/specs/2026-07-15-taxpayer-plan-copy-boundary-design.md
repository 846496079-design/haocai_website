# 纳税人套餐文案边界设计

## 背景与目标

当前官网把一般纳税人的业务流程复杂度扩写为“增强测算、多账户、优先提醒、优先客服”等套餐特权，但现有产品规格没有为这些等级差异提供依据。两个套餐的主要差异是适配的记账报税业务流程不同：一般纳税人流程环节更多、处理更复杂，因此价格不同；核心产品能力不按套餐人为拆分等级。

本轮目标是把公开文案收敛到可核对事实，避免用户把流程差异理解成功能、账号数量、提醒优先级或服务等级差异。

## 可公开表达的套餐差异

- 小规模企业套餐适配小规模纳税人的记账报税流程。
- 一般纳税人套餐适配一般纳税人的记账报税流程。
- 两类流程的处理环节和复杂度不同，因此采用不同价格。
- 已确认价格继续保留：中国站小规模企业 `360 元/年`、`1888 元永久买断`；一般纳税人 `1056 元/年`、`3888 元永久买断`。
- 具体开通范围和服务内容继续以双方确认为准。

## 两个套餐共同展示的核心能力

两个套餐使用相同的核心能力清单，只在第一项标明对应的纳税人业务流程：

1. 适配对应纳税人类型的记账报税流程。
2. 工资、发票、银行流水归集与核对。
3. 凭证生成与财务报表查看。
4. 税额辅助测算与申报资料预览。
5. AI 待办提醒与异常说明。
6. 手机与电脑端查看。

现有产品规格已经确认企业切换与授权协同是产品通用能力，因此在对比表中作为两个套餐共同支持的能力展示，不作为一般纳税人套餐的独占能力。

## 删除或改写的高风险表述

以下表述没有套餐差异依据，应从套餐卡、对比表和相关新闻中删除：

- “包含小规模全部功能”或“进阶套餐”“增强版能力”。
- 税额测算的“基础 / 增强”分级。
- 银行流水的“单账户 / 多账户”分级。
- AI 提醒的“普通 / 优先”分级。
- 客服响应的“普通 / 优先”分级及未定义的响应时效。
- 把多企业账套、授权协同写成一般纳税人套餐独占能力。
- 未提供来源的传统人工代账价格、日均成本及服务模式对比。
- “没有隐藏收费”“保证服务质量”等无法从现有产品规格直接证明的绝对表述。

## 页面与内容调整

### 套餐卡

- 中国站、日本站和香港站均使用相同的信息逻辑：同一组核心能力、不同的适用纳税人流程和价格。
- 一般纳税人套餐不再使用“进阶”“包含全部功能”等表示功能等级差异的标签或文案。
- 套餐副标题明确价格差异来自适配流程及处理复杂度，不暗示产品能力被拆分。

### 功能对比表

- 对比表只比较两个账大师套餐，删除“传统人工代账”列。
- 第一组展示适用纳税人类型、对应业务流程、年度费用和长期方案。
- 后续分组展示共同能力，两个套餐均使用中性的“支持”表述。
- 不使用“实时”“优先”“增强”“单账户 / 多账户”等没有套餐依据的等级词。

### 相关首页与新闻内容

- 删除首页价格摘要中未提供来源的传统人工代账比较。
- 收敛与套餐选择直接相关的绝对承诺和效果承诺。
- “一般纳税人套餐”新闻改为说明适配流程和价格差异，不再宣称增强能力、独占多账户或更高服务优先级。
- 日文、繁体中文稿件同步采用相同边界。

## 不在本轮范围

- 不修改已经确认的套餐价格、购买方式和页面视觉结构。
- 不调整产品功能实现、试用表单、支付或开通流程。
- 不重构与套餐文案无关的页面和组件。
- 不修改工作区中已有的无关改动。

## 验收标准

1. 中国站、日本站、香港站套餐卡只以适用流程和价格区分两个套餐。
2. 三站功能对比表不再包含传统代账列或任何无依据的功能、账号、提醒、客服等级差异。
3. 两个套餐展示的共同核心能力语义一致，仅纳税人流程名称不同。
4. 相关一般纳税人新闻不再出现“进阶、增强、全部功能、多账户独占”等表述。
5. 全局搜索不再命中本规格列出的高风险套餐差异用语；保留的“优先”仅限与套餐等级无关的正常流程说明。
6. 中文内容使用 UTF-8，未出现乱码。
7. `npm run build` 成功，产品页套餐卡与对比表在桌面和移动端均可正常阅读。
