# 飞书 DeepSeek 官网线索智能问数机器人设计

## 1. 背景

官网线索可靠中转已经使用本地 SQLite 保存体验套餐线索和代理合作线索，并具备自动重试、官网后台查看与超过 6 小时预警能力。现有飞书自定义机器人 Webhook 只能主动发送通知，不能接收用户问题，也不能完成智能问数。

本次新增企业自建应用机器人。机器人通过飞书官方 Node SDK 长连接接收单聊或指定群聊中的问题，通过 DeepSeek API 把自然语言转换为受控查询计划，再由官网服务器执行只读汇总查询并回复飞书。机器人同时承接现有线索超时预警，形成统一的官网运营助手。

## 2. 目标

1. 支持用户用自然语言询问官网线索数量、状态、趋势、对比和峰值。
2. 支持类似“上周新增多少代理商线索，多少用户线索”“最近一个月哪天线索最多”“这周比上周多了多少”等自由表达。
3. 使用 DeepSeek API 理解问题，但不允许模型直接访问数据库或生成可执行 SQL。
4. 只返回汇总数据，不返回姓名、电话、公司、备注等个人或业务明细。
5. 仅允许配置的飞书用户和群聊执行问数。
6. 使用同一个应用机器人主动发送超过 6 小时预警和恢复通知。
7. 飞书或 DeepSeek 故障不得影响官网线索接收、持久化和自动转发。

## 3. 非目标

1. 不开放任意 SQL、数据库管理或线索修改能力。
2. 不通过机器人查询单条线索详情或个人信息。
3. 不在本期连接外部 CRM、财务系统、新闻 CMS 或其他业务数据库。
4. 不让 DeepSeek 生成未经校验的业务数字。
5. 不在本期实现销售分配、客户跟进、商机阶段或自动营销。

## 4. 总体架构

生产服务器新增独立 PM2 进程 `zds-feishu-bot`。进程使用飞书企业自建应用的 App ID 和 App Secret，通过官方 Node SDK 建立 WebSocket 长连接，订阅 `im.message.receive_v1` 事件。

消息处理链路如下：

1. 飞书将用户消息推送给 `zds-feishu-bot`。
2. 机器人检查会话和用户是否位于允许名单，并对飞书事件 ID 去重。
3. 机器人把问题、当前北京时间和允许的查询能力说明发送给 DeepSeek。
4. DeepSeek 只返回符合固定 JSON Schema 的查询计划。
5. 服务端验证查询计划，计算可信时间边界，并执行参数化只读聚合查询。
6. 服务端根据真实查询结果生成飞书卡片并回复原会话。

预警链路由现有线索 worker 触发。应用机器人配置完整时，通过飞书消息 API 向指定告警群发送预警；应用机器人未配置时可以临时回退到现有 Webhook。两种通道不得同时发送同一批预警。

## 5. 组件边界

### 5.1 飞书配置模块

负责读取并验证 App ID、App Secret、允许的用户和会话、告警会话，以及 DeepSeek 配置。所有变量只在服务端读取，不使用 `NEXT_PUBLIC_` 前缀。

### 5.2 飞书长连接进程

负责连接、重连、事件接收、事件去重、权限检查和消息回复。事件处理必须快速完成接收确认；DeepSeek 调用和数据库查询在受控处理队列中执行，避免飞书超时重推造成重复回复。

### 5.3 DeepSeek 适配器

使用 OpenAI 兼容的 Chat Completions 接口封装模型调用。适配器输入仅包含用户问题、当前北京时间、时区和查询能力字典；不包含数据库记录、线索正文或密钥。

适配器输出必须通过 JSON Schema 校验。模型超时、限流、网络错误、空响应或非法 JSON 都视为理解失败，不进入数据库查询。

### 5.4 查询计划校验器

查询计划只允许以下能力：

- 指标：线索数量。
- 线索类型：体验套餐、代理合作或全部。
- 状态：待发送、已送达、已暂停、超过 6 小时或全部。
- 时间：今天、昨天、本周、上周、最近若干天、本月、上月及明确起止日期。
- 展示方式：合计、按线索类型分组、按状态分组、按日或按周趋势、两个时间段对比、峰值日期。

校验器拒绝未知字段、未知指标、超过合理上限的时间范围、个人明细请求和任何 SQL 文本。相对时间由服务端按 `Asia/Shanghai` 计算，模型不得决定数据库时区规则。

### 5.5 线索统计模块

统计模块只访问 `lead_submission` 的结构化列：`kind`、`status`、`created_at`、`delivered_at` 和预警相关时间。查询使用参数绑定，不解密 `payload_ciphertext`，因此姓名、电话、公司、城市、备注等字段不会进入问数链路。

“新增线索”按官网成功写入 `lead_submission.created_at` 统计，与是否成功转发无关。重复提交命中现有去重逻辑时不会产生新记录，也不会重复计数。

### 5.6 飞书回复模块

回复使用统一飞书卡片，包含：问题摘要、统计范围、核心数字、必要的分组或对比、数据生成时间和官网后台入口。机器人回复由服务器根据真实查询结果构造，DeepSeek 不负责填充最终数字。

### 5.7 事件去重记录

在现有线索 SQLite 中新增独立的 `feishu_bot_event` 表，只保存唯一事件 ID、脱敏会话和用户摘要、处理状态、查询意图、错误码以及创建和完成时间。事件 ID 使用唯一约束保证同一飞书事件只处理一次。该表不保存完整提问、模型原始响应或最终统计结果；记录当前永久保留，与线索数据采用相同的在线备份流程。

## 6. DeepSeek 查询计划契约

DeepSeek 返回的顶层结构包含版本、意图、指标、时间范围、筛选、分组和对比配置。允许的意图为：

- `SUMMARY`：单个时间范围的数量汇总。
- `COMPARE`：两个时间范围的差值和变化率。
- `TREND`：按日或按周统计序列。
- `PEAK`：寻找指定范围内数量最高的日期或周。
- `HELP`：问题与官网线索问数无关或信息不足。

服务端使用固定枚举和 JSON Schema 严格验证。模型返回的解释性文本只可用于调试，不参与数据库查询，也不直接展示给用户。

## 7. 时间与统计口径

1. 所有业务时间按北京时间计算，数据库边界统一转换为 ISO 时间后查询。
2. 一周从周一 00:00 开始，到下一周周一 00:00 结束。
3. “最近 N 天”采用包含今天的滚动自然日范围，N 必须位于 1 至 366。
4. 未指定时间且询问“新增”时默认查询今天。
5. 询问“当前待发送、暂停或超过 6 小时”时使用实时状态快照，不强制增加创建日期条件。
6. 对比变化率的基期为零时不显示百分比，只显示数量差值。

## 8. 权限与隐私

1. 单聊必须命中 `FEISHU_BOT_ALLOWED_USER_IDS`。
2. 群聊必须命中 `FEISHU_BOT_ALLOWED_CHAT_IDS`，并要求用户明确提及机器人或回复机器人消息。
3. 告警只发送到 `FEISHU_BOT_ALERT_CHAT_ID`。
4. 未授权用户只收到无权查询提示，DeepSeek 和数据库都不会被调用。
5. DeepSeek 输入不得包含线索正文、数据库行、电话号码、公司名、备注、App Secret 或 API Key。
6. 应用日志记录事件 ID、脱敏用户标识、查询计划类型、耗时和错误码，不记录完整问题、模型原始响应或最终业务数据。
7. 飞书 App Secret 和 DeepSeek API Key 只保存在服务器共享 `.env.local`，不得提交到仓库、构建日志或前端资源。

## 9. 配置契约

服务器环境文件预留以下变量：

```text
FEISHU_BOT_APP_ID
FEISHU_BOT_APP_SECRET
FEISHU_BOT_ALLOWED_CHAT_IDS
FEISHU_BOT_ALLOWED_USER_IDS
FEISHU_BOT_ALERT_CHAT_ID

LEAD_QUERY_AI_PROVIDER=deepseek
LEAD_QUERY_AI_API_URL
LEAD_QUERY_AI_API_KEY
LEAD_QUERY_AI_MODEL
```

允许名单使用英文逗号分隔。配置模块必须拒绝空 App ID、过短 Secret、非 HTTPS 的 DeepSeek 地址和不支持的 provider。真实 API 地址、密钥和模型名在用户提供 DeepSeek SDK 或接口资料后写入服务器环境，不写入版本控制。

## 10. 主动预警迁移

1. 应用机器人配置完整并通过启动检查后，预警主通道切换为飞书消息 API。
2. 现有 `LEAD_ALERT_FEISHU_WEBHOOK_URL` 继续保留为迁移期备用。
3. 每次预警只选择一个通道：优先应用机器人，应用机器人不可用时使用 Webhook。
4. 预警仍只包含未送达数量、超过 6 小时数量、最老等待时长和后台链接，不包含线索明文。
5. 预警发送失败继续写入 `lead_alert_delivery`，不改变线索投递状态。

## 11. 异常处理

1. DeepSeek 超时或限流：机器人回复“智能问数暂时不可用，请稍后重试”，不查询数据库。
2. 查询计划非法：机器人返回可理解的失败说明和示例问题，不尝试修补为 SQL。
3. 数据库查询失败：机器人回复查询失败并记录脱敏错误，不返回部分结果。
4. 飞书回复失败：记录事件 ID 和错误码，由事件去重表避免重推后产生多次业务查询。
5. 长连接断开：由 SDK 和 PM2 负责重连与进程拉起；官网网站和 `zds-lead-worker` 保持独立运行。
6. 机器人配置缺失：`zds-feishu-bot` 启动失败并给出明确配置错误，网站和线索 worker 不受影响。

## 12. 部署与进程

发布包增加飞书机器人 worker 和 DeepSeek 适配器。`server-release.sh` 在不输出密钥的前提下检查变量完整性，并通过 PM2 启动或重载 `zds-feishu-bot`。

发布健康检查包含机器人进程在线状态和本地启动自检，但不向 DeepSeek 发送付费测试请求，也不在发布过程中向飞书群发送测试消息。生产验收消息由管理员在飞书中主动发起。

## 13. 测试

自动测试覆盖：

1. 授权用户、未授权用户、授权群和未授权群。
2. 飞书事件重复投递和单次回复。
3. DeepSeek 正常 JSON、非法 JSON、未知字段、SQL 注入文本、超时、限流和网络错误。
4. 今天、上周、最近 N 天、本月、跨月、跨年和北京时间边界。
5. 汇总、按类型分组、状态快照、趋势、对比和峰值。
6. 数据库查询结果与直接 SQL 测试夹具一致。
7. 问数路径不解密线索正文，日志和 DeepSeek 请求不包含测试姓名、电话或备注。
8. 应用机器人预警、Webhook 回退、重复抑制和恢复通知。
9. 飞书机器人、DeepSeek 或预警故障不影响公开表单接收和线索转发 worker。

## 14. 生产验收

1. 在允许的飞书单聊或群聊中询问“上周新增多少代理商线索，多少用户线索”，结果与官网后台数据库统计一致。
2. 询问“这周比上周多了多少”和“最近一个月哪天最多”，机器人返回正确对比和峰值日期。
3. 未授权用户提问时不调用 DeepSeek，也不访问数据库。
4. 关闭 DeepSeek 网络后，机器人明确提示暂时不可用，官网表单仍能正常接收。
5. 构造超过 6 小时未送达线索，指定飞书群收到应用机器人预警且不包含个人信息。
6. 重启服务器后，网站、线索 worker 和飞书机器人三个 PM2 进程均恢复在线。

## 15. 既定产品决策

- 采用企业自建应用机器人和官方 Node SDK 长连接，不使用公网事件回调。
- DeepSeek 负责自然语言理解，不直接访问数据库。
- 后端只执行经过 Schema 校验的受控汇总查询。
- 第一期只查询官网线索库，不查询其他业务系统或个人明细。
- 同一个应用机器人负责智能问数和主动预警，现有 Webhook 仅作迁移备用。
- DeepSeek SDK 或接口资料由用户后续提供，当前先完成不含真实凭据的配置和适配边界。
