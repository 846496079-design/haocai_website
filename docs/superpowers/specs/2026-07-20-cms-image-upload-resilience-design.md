# CMS 图片上传生产容错设计

## 背景与根因

生产 CMS 的 `/api/cms/upload/` 在路由处理函数执行前返回纯文本 `500 Internal Server Error`，而其他 CMS 接口可以正常进入鉴权。上传路由当前在模块顶层静态导入 `sharp`；当 Linux standalone 发布包中的 `sharp` 原生运行时不能加载时，整个路由模块初始化失败，处理函数内部的错误捕获无法生效。编辑器随后把纯文本错误强制按 JSON 解析，最终向用户显示 `Unexpected token`，且发布检查把“当前没有上传任务”错误显示为“图片上传已完成”。

## 目标

- 恢复封面和正文图片上传，不让图片优化组件的单点故障阻断 CMS 上传。
- `sharp` 可用时保留现有压缩、旋转、主图与缩略图生成逻辑。
- `sharp` 不可用或处理失败时保存已通过基础校验的原图，并向前端明确标识降级结果。
- 上传期间继续锁定相关编辑与发布操作，成功、降级和失败都给出可继续操作的中文引导。
- 保留现有文件类型、10 MB 大小限制、对象存储、本地开发存储和素材记录逻辑。

## 变更范围

### 服务端上传处理

上传路由不再顶层导入 `sharp`，而是在图片处理阶段动态加载。路由先完成登录、表单、MIME 类型和文件大小校验，再尝试读取图片元数据并生成优化资源。

处理结果分为两类：

1. `optimized`：`sharp` 正常工作，继续生成最长边不超过 1600 像素的主图和 960 × 540 缩略图；透明图使用 PNG，其他图片使用 JPEG。
2. `original`：`sharp` 加载或处理失败，保存原始 JPEG、PNG 或 WebP 文件；主图 URL 和缩略图 URL 指向同一原图，素材宽高在无法可靠识别时使用兼容占位值。降级只覆盖图片优化故障，文件类型、文件大小、登录、对象存储或数据库错误仍返回失败，不伪装为成功。

接口成功响应新增 `processingMode`，取值为 `optimized` 或 `original`。降级响应同时返回面向用户的说明，便于编辑器提示“图片已保存为原图，可以继续编辑和发布”。

### standalone 发布包

Next.js 输出追踪显式包含上传路由使用的 `sharp`、`@img` 原生包，并在生成发布包时验证 Linux x64 的 `sharp` 运行文件存在。该检查用于修复正常优化路径；原图降级用于防止以后原生运行时异常再次中断上传。

### 编辑器状态与错误引导

- 上传响应先按响应内容类型安全解析；服务端返回纯文本或空响应时，显示“图片服务暂时不可用，请稍后重试”，不暴露 JSON 解析异常。
- 上传状态区分“上传中”“上传成功”“已保存原图”“上传失败”和“尚未上传”，不再把空闲状态等同于成功。
- 封面上传成功后继续写入三语封面字段、标记草稿已修改、要求重新保存和预览。
- 原图降级成功时允许继续保存、预览和发布，并提示后续可重新上传以获得优化图片。
- 上传中继续挂起保存、翻译、预览和发布，避免并发误操作。

## 数据流

1. 用户选择封面或正文图片。
2. 编辑器进入“上传中”，其他关键操作暂时不可用。
3. 服务端校验登录、文件类型和大小。
4. 服务端动态加载 `sharp` 并尝试优化。
5. 优化成功时保存主图和缩略图；优化失败时保存原图。
6. 服务端记录 `cms_asset`，返回素材信息与处理模式。
7. 编辑器更新内容和上传状态，并给出保存草稿、重新预览的下一步引导。

## 错误边界

- 未登录返回 `401` JSON。
- 文件缺失、类型不支持或超过 10 MB 返回 `400` JSON。
- `sharp` 加载或图片处理失败进入原图降级，不返回技术堆栈。
- 对象存储、文件持久化或素材记录失败返回可读 JSON 错误，不能把未持久化文件标记为上传成功。
- 客户端遇到非 JSON 响应时显示稳定的中文故障提示，并将状态标记为失败。

## 验证标准

- 专项测试覆盖优化成功、`sharp` 加载失败后的原图降级、非法格式拒绝、非 JSON 错误解析和上传状态文案。
- CMS 类型检查及既有翻译、发布引导回归通过。
- Linux x64 standalone 发布包包含 `sharp` 及对应 `@img` 原生运行文件，路由加载检查通过。
- 生产部署后，未登录访问 `/api/cms/upload/` 返回鉴权或方法错误 JSON/标准响应，不再返回模块初始化的纯文本 500。
- 登录 CMS 后可上传封面；成功后页面出现封面预览，发布检查不再缺少“新闻封面”。

## 非目标

- 不修改翻译 API、三语发布规则、稿件版本冲突规则或对象存储选型。
- 不新增 S3/MinIO 适配器，不迁移现有图片。
- 不在本次缺陷修复中调整图片编辑、裁剪或批量上传能力。
