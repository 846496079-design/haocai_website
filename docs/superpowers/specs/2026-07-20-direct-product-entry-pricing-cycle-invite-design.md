# 官网立即使用、报价切换与邀请码入口设计

## 背景

当前官网的“立即使用”和“开始试用”会先打开用户信息收集表单，提交联系人、电话和可选邀请码后再进入产品。中国站首页和产品中心的报价区同时展示年付与永久买断四张卡；邀请码还支持从 `/cn/?c=邀请码` 自动跳转到代理合作页。

本次调整取消用户试用信息收集，简化报价展示，并把邀请码识别入口收敛到代理合作页。

## 目标行为

### 立即使用

- 官网所有“立即使用”“开始试用”等面向产品用户的主要按钮，统一进入官网内部路径 `/go/product/`。
- `/go/product/` 根据请求设备自适应跳转：电脑端进入 `https://finance-ai.haocai360.cn/sign-in`，手机端和所有平板设备进入 `https://finance-h5.haocai360.cn/`。
- 设备判断优先使用 `Sec-CH-UA-Mobile`，并结合 User-Agent 识别 iPhone、iPad、iPod、Android 手机与平板、Mobile、Tablet、Silk、Kindle 和常见移动 WebView；无法识别时按电脑端处理。
- 跳转使用临时重定向，响应设置 `Cache-Control: private, no-store, max-age=0`，并设置 `Vary: User-Agent, Sec-CH-UA-Mobile`，避免不同设备之间复用错误的跳转结果。
- 不再展示试用申请弹窗，不再收集联系人、电话或邀请码，不再从公开页面调用用户试用线索提交接口。
- 代理合作页的代理商线索表单继续保留，不受本次调整影响。
- 后端现有用户线索接口暂不删除，避免扩大变更范围；公开官网没有调用入口，处于停用状态。

### 报价方案

- 首页和产品中心共用现有报价组件，顶部增加“年付”和“永久”两个明显切换标签。
- 默认展示“年付”，只显示“小规模企业”和“一般纳税人”两张卡。
- 切换到“永久”后，仍只显示这两个套餐，但价格改为对应永久买断价。
- 中国站小规模企业月度提示从“一天只要一块钱”改为“30 元/月”。
- 中国站一般纳税人月度提示从“≈ 2.9 元/天”改为“88 元/月”。
- 年付价保持 360 元/年和 1056 元/年；永久买断价保持 1888 元和 3888 元。
- 日本站和香港站使用同一切换结构，并显示对应本地化标签和现有价格内容。

### 邀请码

- `/cn/?c=LFFWTL` 不再跳转，也不再自动填充代理合作表单。
- `/cn/partners/?c=LFFWTL` 在代理合作页自动填充邀请码 `LFFWTL`，并在页面完成客户端初始化后直接定位到 `#partner-form` 表单区域，用户仍可修改邀请码。
- 代理合作页继续兼容 `inviteCode`、`c`、`C` 三个查询参数，按原有优先级取第一个非空值。
- 定位时使用 `history.replaceState` 补充 `#partner-form`，不增加浏览器历史记录；随后调用表单区域的即时滚动，并沿用现有 `scroll-mt-24` 避开固定导航栏。
- 未携带有效邀请码参数的 `/cn/partners/` 继续展示代理合作首页，不自动定位表单；邀请码只预填，不自动提交。
- 非代理合作页面不读取邀请码参数。

## 方案比较

### 方案 A：官网统一跳转入口（采用）

所有 CTA 指向同一官网内部路径，由服务端根据设备类型返回目标地址。该方案不依赖页面 JavaScript 或屏幕宽度，支持复制链接和浏览器原生导航；同时通过禁止缓存和 `Vary` 响应头隔离不同设备。

### 方案 B：浏览器 JavaScript 判断

点击时读取 `navigator.userAgent`、触摸点和屏幕宽度后跳转。能够补充少数触屏设备判断，但依赖 JavaScript 完成加载，链接复制和禁用 JavaScript 场景不稳定，因此不采用。

### 方案 C：按响应式断点渲染两个链接

通过 CSS 在不同页面宽度下显示不同链接。电脑缩小窗口会被误判为手机，平板横屏也可能被误判为电脑，因此不采用。

## 组件与数据流

1. 导航、首屏、报价卡、开始使用区和底部 CTA 使用同一个内部产品入口常量 `/go/product/`。
2. 用户点击后，请求官网产品入口路由；路由读取客户端提示和 User-Agent，返回不缓存的 `307` 临时重定向。
3. 电脑端重定向到 AI 财务登录页；手机端、iPad 和安卓平板重定向到 H5 产品页。
4. 跳转不经过官网表单、线索接口或成功弹窗。
5. 报价组件维护当前购买周期状态，只改变两张套餐卡的价格和说明，不改变套餐能力列表。
6. 代理合作页加载时读取当前页面查询参数，预填代理邀请码并将当前地址替换为带 `#partner-form` 的同页地址，再即时滚动到表单区域；提交仍沿用代理线索可靠队列。

## 变更边界

- 不删除用户线索数据库结构、CMS 用户线索页签或用户线索 API。
- 不修改代理商线索提交、可靠队列、工作进程和 CMS 线索管理逻辑。
- 不修改新闻、公司介绍、Logo 或当前工作区中的其他官网文案。
- 不改变产品登录站本身的登录、注册或体验流程。

## 验收标准

1. 电脑端请求 `/go/product/` 时，以 `307` 跳转到 `https://finance-ai.haocai360.cn/sign-in`。
2. 手机、iPad 和安卓平板请求 `/go/product/` 时，以 `307` 跳转到 `https://finance-h5.haocai360.cn/`。
3. 跳转响应禁止缓存并声明按 User-Agent 和客户端移动提示区分。
4. 官网 DOM 中不存在试用用户信息表单，点击按钮不会请求 `/api/public/leads/trial/`。
5. 首页和产品中心默认显示“年付”标签和两张年付卡；切换“永久”后显示两张永久买断卡。
6. 中国站两张年付卡分别显示“30 元/月”和“88 元/月”。
7. `/cn/?c=LFFWTL` 保持在首页；`/cn/partners/?c=LFFWTL` 自动填充邀请码、地址包含 `#partner-form`，桌面端与移动端首屏都定位到表单区域。
8. 代理合作表单仍可提交，CMS 线索管理和新闻动态读取修复不受影响。
